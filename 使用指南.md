# 模拟面试器 - 使用指南

## 🎯 项目概述

这是一个基于 AI 的智能模拟面试系统，提供：
1. **智能简历评分** - 多维度深度分析并提供专业改进建议
2. **个性化面试问题** - 根据简历内容定制专属面试题库
3. **深度答案评估** - 全方位专业评估并附参考答案

## ⚡ 快速开始

### 方式一：Maven 运行（推荐）

```bash
cd mock-interview-agent
mvn spring-boot:run
```

### 方式二：打包后运行

```bash
cd mock-interview-agent
mvn clean package
java -jar target/mock-interview-agent-0.0.1-SNAPSHOT.jar
```

### 访问应用

浏览器打开：http://localhost:8080

## 📋 使用流程

### 步骤 1️⃣: 上传简历

1. 点击首页"开始面试准备"按钮
2. 进入上传页面 (`/upload`)
3. 拖拽或点击选择简历文件
   - 支持格式：PDF、DOC、DOCX、TXT
   - 文件大小：≤ 10MB
   
**测试简历**: 可使用项目根目录下的 `test-resume.txt` 进行测试

### 步骤 2️⃣: 查看简历分析

上传成功后自动跳转到分析页面 (`/analysis/{resumeId}`)

**分析内容包括**:
- 📊 **综合评分** (满分 100 分)
- 📈 **五维详情**
  - 项目经验 (40 分)
  - 技能匹配 (20 分)
  - 内容完整性 (15 分)
  - 结构清晰度 (15 分)
  - 表达专业性 (10 分)
- ✨ **优势亮点** - 简历的优点列表
- 💡 **改进建议** - 具体的优化方案（含优先级）

### 步骤 3️⃣: 开始模拟面试

点击"开始模拟面试"按钮进入面试页面 (`/interview/{resumeId}`)

**面试特点**:
- 🎯 基于简历内容智能生成问题
- 📚 覆盖技术领域：
  - Java 基础、集合、并发
  - Spring/Spring Boot
  - MySQL、Redis
  - 项目经历深度挖掘
- 📊 难度梯度分布：基础 30% + 进阶 50% + 专家 20%

**答题技巧**:
- 采用 STAR 法则（情境、任务、行动、结果）
- 结合具体项目经验
- 突出技术亮点和解决问题能力

### 步骤 4️⃣: 查看评估报告

提交答案后自动跳转到结果页面 (`/result/{resumeId}`)

**评估报告包含**:

#### 📊 总结概览
- 总分及等级（优秀/良好/及格/不及格）
- 整体反馈评价
- 分类得分统计（按技术领域）
- 优势与改进建议

#### 📝 详细评估
- 逐题评估：
  - 原始问题
  - 你的回答
  - 本题得分
  - 具体反馈（优点 + 不足）

#### 📚 参考答案
- 每道题的深度解析版答案
- 核心要点列表
- 最佳实践建议

## 🛠️ 技术配置

### API Key 配置

在 `src/main/resources/application.yml` 中配置：

```yaml
spring:
  ai:
    dashscope:
      api-key: your-api-key-here  # 替换为你的通义千问 API Key
```

### 环境变量方式（推荐）

```bash
# Windows PowerShell
$env:AI_DASHSCOPE_API_KEY="your-api-key-here"
mvn spring-boot:run

# Linux/Mac
export AI_DASHSCOPE_API_KEY="your-api-key-here"
mvn spring-boot:run
```

## 🔍 核心功能说明

### 1. 简历评分系统

**评分维度**:
- **项目经验 (40 分)**: 技术深度、业务价值、量化成果
- **技能匹配 (20 分)**: 技术栈专业度、核心能力
- **内容完整性 (15 分)**: 模块顺序合理性
- **结构清晰度 (15 分)**: 技术名词规范性
- **表达专业性 (10 分)**: 语言简洁性

**输出内容**:
- 总体评分
- 各维度得分
- 优势分析
- 改进建议（含优先级）

### 2. 面试问题生成

**问题类型**:
- PROJECT (项目经历)
- JAVA_BASIC (Java 基础)
- JAVA_COLLECTION (Java 集合)
- JAVA_CONCURRENT (Java 并发)
- MYSQL (数据库)
- REDIS (缓存)
- SPRING (Spring 框架)
- SPRING_BOOT (Spring Boot)

**出题原则**:
- 与简历高度相关
- 难度梯度分布
- 考察深度理解
- 注重实战经验

### 3. 答案评估系统

**评估维度**:
- 准确性 (40%) - 技术概念正确性
- 完整性 (20%) - 核心知识点覆盖
- 深度 (25%) - 底层原理理解
- 表达 (15%) - 逻辑清晰度

**评分标准**:
- 90-100: 优秀（源码级理解）
- 75-89: 良好（概念正确完整）
- 60-74: 及格（核心概念正确）
- 40-59: 不及格（明显错误）
- 0-39: 较差（答非所问）

## ⚠️ 注意事项

1. **文件格式**: 目前仅支持文本格式的简历文件
2. **内存存储**: 数据存储在内存中，重启后会丢失
3. **API 调用**: 需要稳定的网络连接访问通义千问 API
4. **响应时间**: AI 处理需要一定时间（通常 10-30 秒），请耐心等待
5. **BrowserTool**: 已暂时禁用浏览器工具（避免 ChromeDriver 依赖问题）
